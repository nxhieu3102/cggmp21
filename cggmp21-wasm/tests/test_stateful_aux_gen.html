📋 Protocol Information

This test demonstrates the auxiliary generation protocol which creates shared auxiliary information needed for the CGGMP21 threshold signature scheme.

Protocol Flow:
  1. Round 1: Generate commitments to random values and ring-pedersen parameters
  2. Reliability Check: Optional round to ensure reliable broadcast (if enforced)
  3. Round 2: Reveal decommitments with actual values
  4. Validation: Validate decommitments and ring-pedersen parameters
  5. Round 3: Generate zero-knowledge proofs
  6. Finalization: Combine all information to create auxiliary data
