📋 Protocol Overview

This test demonstrates the hierarchical threshold secret sharing (HTSS) key generation protocol, an advanced variant of the CGGMP21 threshold signature scheme that provides fine-grained access control through party rankings.

Protocol Flow:
  1. Round 1: Generate commitments using rank information
  2. Reliability Check: Optional verification of broadcast channel integrity
  3. Round 2: Broadcast decommitments and distribute rank-based polynomial shares
  4. Round 3: Generate and verify Schnorr proofs with rank constraints
  5. Finalization: Create hierarchical threshold key shares with preserved rank information

🏛️ Hierarchical Threshold Secret Sharing (HTSS)

Key Concept: Unlike traditional threshold schemes where any t out of n parties can recover the secret, HTSS assigns ranks to parties that determine their access privileges.

Access Control Rules:

✅ Valid Access: For threshold t=3, parties with ranks (0,1,1) can recover the secret
✅ Valid Access: For threshold t=3, parties with ranks (0,1,2) can recover the secret
❌ Invalid Access: For threshold t=3, parties with ranks (1,1,2) cannot recover the secret

Rule: For any t shares with ranks (r₁, r₂, ..., rₜ) sorted by rank, recovery is possible if and only if rᵢ ≤ i-1 for all i ∈ {1, 2, ..., t}

📚 Educational Notes:
• Hierarchical threshold schemes provide more granular access control than traditional threshold schemes
• Ranks determine the "priority" or "level" of each party in the hierarchy
• Lower ranks have higher privileges (rank 0 > rank 1 > rank 2, etc.)
• The polynomial evaluation uses derivatives based on ranks for enhanced security
• This implementation maintains compatibility with the existing CGGMP21 infrastructure
